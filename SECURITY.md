# Security Policy

If you believe you have found a security issue, report it privately instead of opening a public issue.

## How To Report

- Use a GitHub Security Advisory when available: [Report a vulnerability](https://github.com/jklock/splitshot/security/advisories/new)
- Include the affected version, platform, steps to reproduce, and expected impact.
- Do not publish exploit details until the issue has been reviewed.

## What To Include

- The exact repository state or commit if known.
- The input that triggers the issue.
- Any logs, screenshots, or crash output that help reproduce the problem.
- Whether the issue affects analysis, export, file handling, or browser UI behavior.
