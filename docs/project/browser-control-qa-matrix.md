# Browser Control QA Matrix

This matrix is the source of truth for browser-visible controls and their current QA coverage.

It is not a claim that every button or field has its own direct behavior test. Rows marked `static`, `smoke`, or `presence-only` are still gaps that should be upgraded when those controls change.

For the exhaustive per-control worklist that complements this matrix, see [browser-control-coverage-plan.md](browser-control-coverage-plan.md).

## How To Use This

Before changing a browser control, find its row here and update the matching tests in the same change.

If a control is missing from this matrix, it does not have an explicit owner yet.

## Known Gaps

- This matrix does not yet give every individual field a dedicated end-to-end test.
- The Project / import row now covers the simplified PractiScore helper-button plus local file-import path, but chooser/open/save flows still rely more heavily on lifecycle and controller contract tests than browser-only end-to-end coverage.
- The metrics pane now has direct downstream coverage for timing-event ordering and scoring-row edit or restore propagation, but save or reopen and export parity are still incomplete.
- The browser interaction file now covers marker import-selected-shot seek, marker playback-window seek or loop wrap, unified marker-list selection and seek, selected-marker workbench launch, workbench open or close flow, editor duplicate or remove rerender, workbench editor continuity, collapsible marker editor sections, direct badge drag base placement, base-drag motion-path preservation, timer badge background color-picker live preview and close-commit, timer/draw/score badge position inputs and lock-to-stack controls, overlay bubble size override, font size, bold/italic controls, export log modal open/close/backdrop and download, marker template defaults for fresh shot-linked markers, collapsed marker navigation, waveform expand/zoom/amplitude, waveform pan and shot movement, overlay visibility and badge toggles, review source-switch after-final render, review custom placement or size, review stack-lock editor behavior, review text-box creation and drag, and review text-box background/text/opacity controls.
- The Settings row is still gap-heavy: section collapse plus import-current/reset-defaults now have direct browser coverage, but most template fields remain presence-tested.
- Some rows are still mostly static or smoke coverage; those are the first candidates for stronger behavior tests.
- When a browser control regresses, update the matrix row and the test in the same change so the coverage claim stays honest.
