# Browser Compact Tool UI Gap Analysis

## Current Gaps

- `Local review cockpit`, `Start here`, and long description copy create clutter.
- `Open Stage Video` appears twice in a large layout before any video is selected.
- Tool rail uses code labels and a large brand block.
- Fonts and metric values are too large for an editing tool.
- Static UI tests do not guard against duplicate instructional text returning.

## Fixes

- Collapse the start state to a single compact action panel.
- Use one-word tool labels in the rail.
- Reduce rail and typography scale.
- Add static UI tests for compact rail and reduced copy.
